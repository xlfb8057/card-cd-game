# 微信小游戏构建与发布检查清单

## 一、构建前（Cocos Creator）

- [ ] **设计分辨率**：720 × 1280，适配宽度 Fit Width
- [ ] **参与构建的场景**：`Battle`、`shop`（项目设置 → 参与构建的场景）
- [ ] **启动场景**：`Battle`
- [ ] **微信 AppID**：在构建面板填入（测试可用游客 / 测试号）
- [ ] **设备方向 / Orientation**：**Portrait 竖屏**
- [ ] **分包 Bundle `later-assets`**：
  - 资源管理器 → `assets/subpackages/later-assets`
  - 已勾选 **配置为 Bundle**，名称 `later-assets`
  - 构建面板 → 该 Bundle **压缩类型** = **小游戏分包 / Mini Game Subpackage**

## 二、Cocos 构建步骤

1. 菜单 **项目 → 构建发布 / Project → Build**
2. **发布平台 / Platform**：**微信小游戏 / WeChat Mini Game**
3. **构建任务名称**：`wechatgame`（默认）
4. 填写 **AppID**（替换 `build-templates` 里的 `touristappid`）
5. **Orientation**：Portrait
6. 点击 **构建 / Build**，完成后 **运行 / Run**（可选）

构建产物目录：`build/wechatgame/`

## 三、构建后自动检查

在项目根目录执行：

```bash
node tools/check-wechat-bundle-size.mjs
```

或指定路径：

```bash
node tools/check-wechat-bundle-size.mjs build/wechatgame
```

验收：

- [ ] 主包 **< 4 MB** ✅
- [ ] 主包 + 全部分包 **< 20 MB** ✅

## 四、微信开发者工具

1. 打开 **微信开发者工具**
2. **导入项目** → 选择 `build/wechatgame` 目录
3. AppID：与构建时一致
4. **详情 → 本地设置 → 调试基础库**：≥ **2.1.0**（支持分包）
5. 点击 **编译 / 预览**

验收：

- [ ] 编译无红色报错
- [ ] 竖屏显示正常
- [ ] 顶部 HUD 不被胶囊按钮遮挡（已挂 `SafeAreaAdapter` 时）
- [ ] 能完整跑一局：战斗 → 商店 → 下一回合
- [ ] 存档：退出再进，数据仍保留（微信 Storage）

## 五、Canvas 组件（编辑器手动一次）

在 **Battle.scene** 和 **shop.scene** 的 **Canvas** 上：

- [ ] 添加 **SafeAreaAdapter** 组件
- [ ] **Hud Root** 拖入 `HUD` 节点（有则填，无则留空）

## 六、配置文件说明

| 文件 | 作用 |
|------|------|
| `build-templates/wechatgame/game.json` | 竖屏、`showStatusBar: false`，构建时合并 |
| `build-templates/wechatgame/project.config.json` | 开发者工具项目配置模板 |
| `assets/scripts/platform/WeChatBridge.ts` | wx.login / 分享 / 广告预留 |
| `assets/scripts/utils/StorageAPI.ts` | 优先 wx.setStorageSync |
| `assets/scripts/platform/SubpackageLoader.ts` | 第 4 回合预加载 later-assets |
