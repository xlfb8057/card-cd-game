# 《大巴扎：像素乱斗》配置数据说明 v1.0

> 描述 `assets/resources/config/` 下 JSON 配置表的字段、约束与修改规范。  
> 类型定义见 `assets/scripts/config/*.ts`。

---

## 1. 配置文件总览

| 文件 | 路径 | 说明 |
|------|------|------|
| items.json | `resources/config/items.json` | 装备定义（15 件） |
| heroes.json | `resources/config/heroes.json` | 英雄定义（3 人） |
| enemies.json | `resources/config/enemies.json` | 敌人定义（5 个） |

**加载方式**：运行时由 `ConfigTable` 通过 Cocos `resources.load`；测试环境用 `MemoryConfigLoader` 注入相同结构。

**修改规则**：

1. 修改 JSON 后须同步 `ItemConfig.ts` / `HeroConfig.ts` / `EnemyConfig.ts` 类型（如有新字段）
2. 修改 `id` 时须同步 `RoundConfig.ts` 中的敌人映射、商店池逻辑
3. 禁止在 System 代码中硬编码装备/敌人数值

## 2. items.json

### 2.1 根结构

```json
{
  "items": [ /* IItemConfig[] */ ]
}
```

### 2.2 单条装备字段（IItemConfig）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | ✅ | 唯一标识，snake_case，如 `pencil_sharpener` |
| name | string | ✅ | 显示名称 |
| rarity | string | ✅ | `bronze` \| `silver` \| `gold` \| `legendary` |
| baseCD | number | ✅ | 基础冷却（秒），实际 CD 受加速与角色被动影响 |
| price | number | ✅ | 商店购买价（金币） |
| tags | string[] | ✅ | 标签，用于角色被动匹配（如 `tool`, `food`, `dot`） |
| effects | IItemEffect[] | ✅ | 效果列表，见下表 |
| heroAffinity | string | ❌ | 推荐英雄 id（`jules`/`mak`/`stelle`） |
| maxStar | number | ✅ | 最大星级，MVP 固定 3（升星系统 Phase 2 启用） |

### 2.3 效果字段（IItemEffect）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | ✅ | 见「效果类型枚举」 |
| value | number | ✅ | 数值含义因 type 而异 |
| target | string | ✅ | `enemy` \| `self` \| `adjacent` \| `all_tools` \| `all_food` |
| trigger | string | ❌ | `onTrigger`（默认）\| `passive` |
| dotType | string | ❌ | 仅 `type=dot` 时：`poison` \| `burn` |

### 2.4 效果类型枚举（EffectType）

| type | value 含义 | 示例 |
|------|-----------|------|
| damage | 单次伤害 | `{ "type": "damage", "value": 15, "target": "enemy" }` |
| dot | 每秒持续伤害 | value=8, dotType=poison |
| shield | 护盾值 | target=self |
| heal | 治疗量 | target=self |
| haste | 减少目标 CD（秒） | target=adjacent, trigger=onTrigger |
| crit | 暴击倍率 | value=2.0 表示 2 倍 |
| freeze | 冻结时长（秒） | target=enemy |

### 2.5 示例：完整装备

```json
{
  "id": "spring_fist",
  "name": "弹簧拳",
  "rarity": "bronze",
  "baseCD": 3.0,
  "price": 5,
  "tags": ["damage", "tool"],
  "effects": [
    { "type": "damage", "value": 25, "target": "enemy" },
    { "type": "haste", "value": 1.0, "target": "adjacent", "trigger": "onTrigger" }
  ],
  "heroAffinity": "stelle",
  "maxStar": 3
}
```

### 2.6 稀有度与商店池（代码逻辑）

商店按当前回合在 `ShopSystem` 中加权随机，大致规则：

| 回合段 | 青铜 | 白银 | 黄金 | 传说 |
|--------|------|------|------|------|
| 1–2 | 100% | — | — | — |
| 3–4 | 70% | 30% | — | — |
| 5 | 50% | 40% | 10% | — |

新增装备只需保证 `rarity` 正确即可进入对应池。

## 3. heroes.json

### 3.1 根结构

```json
{
  "heroes": [ /* IHeroConfig[] */ ]
}
```

### 3.2 单条英雄字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | ✅ | `jules` / `mak` / `stelle` |
| name | string | ✅ | 显示名 |
| maxHP | number | ✅ | 最大生命值，MVP 均为 100 |
| passive | object | ✅ | 被动技能，见下表 |
| activeSkill | object | ✅ | 主动技能，见下表 |

### 3.3 passive.type 枚举

| type | 字段 | 说明 |
|------|------|------|
| food_boost | value, overflowToShield | 食物效果 +value，治疗溢出转盾比例 |
| dot_boost | value | DOT 伤害 +value（0.3 = +30%） |
| tool_cd | value | 工具类装备 CD 减少 value（0.2 = -20%） |

### 3.4 activeSkill 字段

| 字段 | 类型 | 说明 |
|------|------|------|
| name | string | 技能名 |
| effect | string | `trigger_all_food` / `adrenaline` / `overload` |
| duration | number | 持续秒数 |
| usesPerBattle | number | 每场可用次数 |

### 3.5 示例

```json
{
  "id": "stelle",
  "name": "斯黛拉",
  "maxHP": 100,
  "passive": { "type": "tool_cd", "value": 0.2 },
  "activeSkill": {
    "name": "过载运转",
    "effect": "overload",
    "duration": 8,
    "usesPerBattle": 1
  }
}
```

## 4. enemies.json

### 4.1 根结构

```json
{
  "enemies": [ /* IEnemyConfig[] */ ]
}
```

### 4.2 单条敌人字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | ✅ | 唯一 id，与 RoundConfig 映射一致 |
| name | string | ✅ | 显示名 |
| hp | number | ✅ | 最大生命 |
| attackCD | number | ✅ | 攻击间隔（秒） |
| damage | number | ✅ | 单次攻击伤害 |
| timeLimit | number | ❌ | 战斗限时（秒），BOSS 用 60 |
| reward | object | ✅ | 胜利奖励，见下 |

### 4.3 reward 字段

| 字段 | 类型 | 说明 |
|------|------|------|
| gold | number | 基础金币（另有 RoundConfig 回合奖励） |
| items | string[] | 掉落占位，如 `bronze_random`（MVP 未完全实现掉落 UI） |

### 4.4 回合映射（RoundConfig.ts）

```typescript
ROUND_ENEMY_IDS = {
  1: 'slime',
  2: 'goblin',
  3: 'shadow_wolf',
  4: 'mage',
  5: 'boss_golem',
};
```

修改敌人出现顺序时**必须**同步此文件。

### 4.5 示例：BOSS

```json
{
  "id": "boss_golem",
  "name": "岩石巨像",
  "hp": 500,
  "attackCD": 1.5,
  "damage": 30,
  "timeLimit": 60,
  "reward": { "gold": 20, "items": ["legendary_random"] }
}
```

## 5. 代码中的流程常量（非 JSON，但属「配置」）

## 6. 新增内容检查清单

添加新装备时：

- [ ] `id` 全局唯一
- [ ] `rarity` 与商店池匹配
- [ ] 至少 1 个 `effects` 条目
- [ ] `baseCD` ≥ 0.3 或接受加速后触底
- [ ] 运行游戏验证商店可刷出

添加新敌人时：

- [ ] 在 `RoundConfig.ROUND_ENEMY_IDS` 登记回合
- [ ] BOSS 建议设 `timeLimit: 60`
- [ ] HP/伤害与 5 回合曲线协调

添加新英雄时：

- [ ] `HeroSystem` 中实现 passive.type / activeSkill.effect
- [ ] `GameBootstrap` 或角色选择 UI 传入 heroId
