## 1. 游戏简介

《大巴扎：像素乱斗》是一款**全自动战斗**的 Roguelike 构筑游戏：战前购买装备、调整 Build，战斗中装备按冷却时间自动触发，玩家可使用少量**救场技能**干预。

- **平台**：浏览器预览 / 微信小游戏（竖屏）
- **单局流程**：5 个回合，每回合 = 战斗 → 商店 → 下一战
- **最终目标**：第 5 回合击败 BOSS「岩石巨像」

---

## 2. 如何运行

### 2.1 Cocos Creator 预览（推荐开发调试）

1. 安装 **Cocos Creator 3.8.8**
2. 打开项目目录：`NewProject`
3. 确认启动场景为 `assets/scene/Battle.scene`
4. 点击编辑器顶部 **预览 ▶**
5. 使用 Chrome 或内置浏览器，竖屏窗口约 720×1280

### 2.2 微信小游戏

详细步骤见 `Doc-BUILD/微信小游戏构建与发布检查清单.md`。

简要步骤：

1. Cocos → **项目 → 构建发布** → 平台选 **微信小游戏**
2. 填入 AppID，方向选 **竖屏 Portrait**
3. 构建完成后，用 **微信开发者工具** 导入 `build/wechatgame`
4. 点击编译运行

## 4. 玩法规则

### 4.1 战斗

- 战斗**全自动**，装备 CD 归零时触发效果（伤害、治疗、护盾、加速等）
- 玩家 HP 归零则失败；敌人 HP 归零则胜利
- 第 5 回合 BOSS 有 **60 秒**限时（普通战斗约 30 秒）
- 回合间在商店可恢复约 **60% 最大生命**

### 4.2 装备

- 最多 **6 件**装备同时上阵
- 装备有**冷却时间（CD）**，相邻装备可互相**加速**
- CD 最低 **0.3 秒**（再快也不会更快）
- 品质：青铜 → 白银 → 黄金 → 传说（商店随回合解锁）

### 4.3 救场技能

| 技能 | 效果 | 限制 |
|------|------|------|
| 紧急充能 | 为一装备充能 50% CD | 每场 2 次，10 秒全局冷却 |
| 手动超载 | 立即释放角色主动技能 | 每场 1 次 |
| 时停调整 | 暂停 3 秒，可交换 2 件装备位置 | 第 6 回合后解锁（5 回合 MVP 中不可用） |

### 4.4 经济

- 每回合基础工资 **5 金币**
- 连胜/连败有额外金币奖励
- 出售装备得买入价 × **0.7**（向下取整）

### 4.5 五个回合敌人

| 回合 | 敌人 | 类型 |
|------|------|------|
| 1 | 史莱姆 | 教学 |
| 2 | 哥布林 | 普通 |
| 3 | 暗影狼 | 普通 |
| 4 | 法师 | 精英 |
| 5 | 岩石巨像 | BOSS（500 HP） |

## 5. 结算与按钮

战斗结束后会显示结算面板：

| 结果 | 可用按钮 | 行为 |
|------|----------|------|
| **胜利** | 进入商店 | 跳转商店，购买后继续 |
| **失败** | 重试 | 回到本回合战前，保留装备与金币 |
| **失败** | 从头再来 | 清空存档，从第 1 回合重新开始（教学装备） |
| **第 5 回合胜利** | — | 显示通关，本局结束 |

> **提示**：重试依赖战前快照；若重试无效，请确认使用的是最新版本。

## 7. 已知限制（MVP）

- 无升星、无改装、无事件节点
- 无角色选择、无背包拖拽换装备 UI（购买自动进栏）
- 美术为白盒占位（文字 + 色块）
- 无存档续玩弹窗（有本地存档，但无「继续游戏」入口）
- BOSS 难度较高，需合理 Build 或多次重试

## 8. 常见问题

**Q：预览黑屏？**  
A：确认启动场景为 `Battle.scene`，且 Canvas 上已挂 `GameBootstrap` 与 `BattleSceneView`。

**Q：商店金币显示不对？**  
A：从战斗胜利进入商店后会自动刷新；若仍异常，返回战斗再进一次。

**Q：微信构建后无法运行？**  
A：检查 AppID、竖屏配置、首包大小（`npm run check:wechat-size`）。

**Q：Cocos 报 EPERM 临时文件错误？**  
A：关闭编辑器，删除项目下 `temp/` 文件夹后重启。
